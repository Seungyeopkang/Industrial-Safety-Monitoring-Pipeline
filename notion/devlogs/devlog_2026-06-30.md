### 기술 결정 사항
- **Hansung-Cho/yolov8-ppe-detection 모델 채택**:
  - *이유*: 실시간 추론 속도(**190.7 FPS**)와 뛰어난 리소스 효율성(파라미터 **3.01 M**, RAM **50MB**), 그리고 Construction-PPE 기준 가장 높은 정확도(**Helmet F1: 0.793, Vest F1: 0.698**)를 기록함.
  - *라벨 구조적 강점*: `NO-Hardhat`, `NO-Safety Vest`, `NO-Mask`와 같은 미착용(위반) 클래스가 자체 탑재되어 있어, 복잡한 룰 엔지니어링(Overlap/IoU 연산 등) 없이 직관적으로 VLM 분기 트리거 신뢰도를 얻을 수 있음.
- **신뢰도 기반 하이브리드 VLM 트리거 설계 확정**:
  - *이유*: 평가 결과 단독 탐지 모델의 미착용(no_helmet) 감지 F1 스코어가 상대적으로 저조(0.08 ~ 0.21)함을 확인. 이는 단독 객체 탐지기가 원거리, 앵글 사각지대, 객체 가림 상황에서 미착용을 판단하는 데 한계가 있음을 보여줌. 
  - *동작 방식*: 1차 탐지 신뢰도가 확실히 높은 정상 상태는 VLM을 건너뛰어 비용을 아끼고, 신뢰도가 애매한 경계 구간(0.3 ~ 0.75)만 선별하여 원본 프레임 및 메타데이터를 VLM으로 전송해 최종 교정 및 보고서를 작성하는 하이브리드 구조가 타당함을 입증함.

### 문제 & 해결
1. **문제**: HuggingFace의 3종 YOLO 모델별로 학습 데이터의 클래스 ID 순서 및 클래스명이 불일치하여, 단순 Validation 실행 시 mAP 및 F1 메트릭 연산 왜곡 발생.
   - *원인*: 각 모델 배포처마다 상이한 PPE 라벨 스키마 및 가중치를 사용함.
   - *해결*: 모델 고유 클래스명을 표준 클래스(`helmet`, `no_helmet`, `vest`)로 통일해주는 매핑 딕셔너리(`MODEL_MAPPINGS`)를 구축하고, 테스트 이미지 단위로 IoU(>=0.5) 기반 매칭을 직접 연산하는 코드를 작성하여 일관적인 Precision/Recall/F1을 도출함.
2. **문제**: Notion API를 활용해 본문 데이터를 업로드할 때 rich_text 내 `annotations` 필드의 잘못된 계층 구조로 인해 400 Validation Error 발생.
   - *원인*: `annotations`를 `text` 오브젝트 하위 필드로 오인해 삽입함.
   - *해결*: `annotations`는 `text`와 형제(sibling) 계층 구조에 위치해야 하므로 API 스펙에 맞게 payload 딕셔너리를 수정하여 정상 연동 성공.
